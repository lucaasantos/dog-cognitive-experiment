# Dog Cognitive Experiment

Dog Cognitive Experiment is a desktop cognitive AI experiment built with Python and PySide6. It simulates memory-based dog-experience adaptation with a hybrid symbolic + machine learning backend, then visualizes the agent as a futuristic cognitive observability dashboard.

Version 3 keeps the existing backend architecture intact while replacing the old Tkinter interface with a dark neural monitoring system. The app now feels less like a static classifier and more like a live adaptive intelligence experiment: experiences stream in, beliefs move, model confidence updates, memory evolves, and learned language patterns become visible.

This is not a chatbot, AGI, or conscious AI. It is a transparent cognitive simulation focused on memory, belief confidence, emotional weighting, symbolic rules, local ML classification, and explainable state changes.

## Features

- Futuristic PySide6 desktop dashboard
- Persistent JSON memory
- Hybrid symbolic + machine learning cognition
- Rule-based event, breed, and context detection
- CountVectorizer text features
- Multinomial Naive Bayes classification
- Prediction confidence from model probabilities
- Automatic memory updates after each experience
- Dataset append and model retraining from the dashboard
- Live learning stream with detected features and belief impact
- Animated fear, confidence, and adaptation stability meters
- Circular neon cognitive gauges
- Top learned token panels for risk and safety classes
- Dataset statistics and class distribution display
- Cognitive activity log and reflection stream
- Explainable ML probability and token evidence panels
- No database, no LLM, and no web framework

## V3 Interface

The PySide6 interface is designed as an AI laboratory dashboard:

- Left side: experience input, current belief, fear meter, confidence meter, adaptation stability, and reflection.
- Right side: live learning stream, ML activity, learned tokens, dataset statistics, prediction confidence, and cognitive activity log.
- Header: `DOG COGNITIVE EXPERIMENT V3` with neon styling and model status feedback.

When a new dog-related experience is submitted, the dashboard:

1. Processes it through the existing cognitive backend.
2. Applies symbolic event/context/breed rules.
3. Reads the Naive Bayes prediction and confidence.
4. Updates persistent memory.
5. Appends the new classified example to `ml/dataset.csv`.
6. Retrains the CountVectorizer + Naive Bayes pipeline.
7. Refreshes gauges, learned tokens, dataset statistics, reflection, and activity logs.

## Installation

Clone the repository:

```bash
git clone https://github.com/your-username/dog-cognitive-experiment.git
cd dog-cognitive-experiment/dog_experiment
```

Optional: create and activate a virtual environment.

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

macOS/Linux:

```bash
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

On Windows, `requirements.txt` uses `PySide6-Essentials` because the dashboard only needs Qt Core, Gui, and Widgets. This avoids the very deep file paths created by the full PySide6 Addons package.

## Training The Model

Train the local classifier:

```bash
python ml/train_model.py
```

The training script:

1. Loads `ml/dataset.csv`
2. Converts text into word-count features with `CountVectorizer`
3. Trains a `MultinomialNB` classifier
4. Saves the trained pipeline as `ml/model.pkl`

## Running

Start the PySide6 dashboard:

```bash
python main.py
```

On Windows, if `python` is not available but the Python launcher is installed:

```bash
py -B main.py
```

## How V3 Works

The application reads a user experience such as:

```text
The dog suddenly ran toward me barking loudly.
```

It then:

1. Normalizes the text for symbolic rule detection
2. Detects weighted events
3. Detects breed multipliers
4. Detects contextual modifiers
5. Classifies the full text with the ML model
6. Reads the model's prediction confidence
7. Combines symbolic impact with ML impact
8. Applies emotional decay
9. Updates persistent memory
10. Appends the classified experience to the local dataset
11. Retrains the local model
12. Recomputes belief confidence and current assessment
13. Refreshes live meters, token panels, dataset statistics, and reflections

The original symbolic impact is still calculated as:

```text
(event weights + context modifiers) * breed multiplier
```

V3 keeps the ML adjustment:

```text
hybrid impact = rule impact + (label emotional weight * prediction confidence)
```

For example, an `aggressive` or `threatening` prediction increases fear and reinforces risk-oriented beliefs. A `friendly`, `playful`, or `calm` prediction reduces fear and weakens confidence in dangerous-belief patterns.

## Explainability

The dashboard exposes the internal cognition loop:

- Active tokens detected from the current input
- Predicted class and probability confidence
- Top learned tokens from Naive Bayes feature log probabilities
- Fear and belief shifts after each memory update
- Dataset growth and class distribution
- Cognitive reflections generated from recent memory

This keeps the project aligned with its purpose: an experimental cognitive agent that adapts through memory, belief confidence, emotional weighting, and experience-based interpretation.
